local-search-mcp — OpenCode MCP 集成指南

本文档说明如何将 local-search-mcp（学术论文搜索 + 深度研究引擎）配置为 OpenCode 的 MCP 服务器。

前提

  • Docker 部署运行中：docker compose up -d --build
  • 服务监听 http://localhost:${PORT:-8765}
  • OpenCode 已安装（建议 v0.14+）

配置方式（二选一）

方式 A：远程连接（推荐 — 适用于 Docker 部署）

OpenCode 通过 HTTP 连接到你运行中的 Docker 容器。需要将配置写入 OpenCode 的全局配置或项目根目录的 opencode.json。

方案 A1：MCP Streamable HTTP 端点（完整 MCP 协议支持）

{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "local-search": {
      "type": "remote",
      "url": "http://localhost:${PORT:-8765}/mcp-stream",
      "enabled": true
    }
  }
}

方案 A2：自定义 JSON-RPC 端点（轻量，仅 tools/list + tools/call）

{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "local-search": {
      "type": "remote",
      "url": "http://localhost:${PORT:-8765}/mcp",
      "enabled": true
    }
  }
}
说明：
  • /mcp-stream 使用 MCP SDK 的 StreamableHTTPServerTransport，支持完整的初始化协商和能力发现。
  • /mcp 是自定义的 JSON-RPC 处理器，仅实现 initialize、tools/list、tools/call、resources/list。对大部分使用场景已足够。

方式 B：本地命令（stdio 模式 — 无需 Docker）

OpenCode 直接启动 Node.js 进程。需要项目依赖已安装（npm install）。

{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "local-search": {
      "type": "local",
      "command": ["node", "/path/to/local-search-mcp/src/mcp_server.js"],
      "enabled": true,
      "timeout": 120000
    }
  }
}
限制：stdio 模式下的 mcp_server.js 仅暴露核心工具（search_web、fetch_page、search_and_fetch、research_problem、get_artifact、engine_status）。论文工具（search_papers、research_deep 等）和缓存工具在 stdio 模式下不可用。如需完整功能，请使用 Docker + 远程连接（方式 A）。

可用工具

工具名称说明需要 ENABLE_PAPER_TOOLS
search_web通用网页搜索（Tavily / Google / Bing）否
fetch_page抓取网页内容（Playwright 浏览器）否
search_and_fetch搜索 + 自动抓取否
research_problem研究问题（多轮搜索 + 分析）否
get_artifact获取保存的 artifact否
engine_status引擎状态否
search_papers学术论文搜索是
lookup_paper论文详情查询是
expand_paper_citations论文引用展开是
find_open_accessOA 版本查找是
research_papers多轮论文研究是
research_deep深度研究（含全文本获取能力）是
locate_paper_content定位论文全文本 URL是
fetch_paper_content获取并提取论文全文本段落是
get_paper_sections按 paper_key 读取缓存的章节是
paper_cache_stats论文缓存统计否（独立启用）
paper_cache_cleanup论文缓存清理（默认 dry-run）否（独立启用）
注意：paper_cache_stats 和 paper_cache_cleanup 不需要 ENABLE_PAPER_TOOLS=true，它们依赖 PAPER_CACHE_ENABLED=true（默认即启用）。所有其他论文工具需要 ENABLE_PAPER_TOOLS=true。

验证配置

配置完成后，启动 OpenCode 并执行：

/status

在输出的 MCP Servers 段中应能看到 local-search 及其工具列表。也可以直接调用：

使用 search_web 搜索 "quantum computing"

或（如果启用了论文工具）：

使用 search_papers 搜索 "transformer attention"

常见问题

OpenCode 看不到工具列表

  • 确认 docker compose ps 显示容器正常运行
  • 确认 curl http://localhost:8765/health 返回 {"ok":true}
  • 检查 OpenCode 配置文件中 mcp 段格式正确（JSON 语法错误）
  • 尝试将配置放在 ~/.config/opencode/opencode.json 全局配置中而非项目配置

论文工具返回 "unknown tool"

  • 确认 .env 中设置了 ENABLE_PAPER_TOOLS=true
  • 重启 Docker 容器：docker compose restart
  • 检查容器日志：docker compose logs 应看到 "Paper sources requiring config" 相关提示

research_deep 超时

  • OpenCode 默认 MCP 超时为 30 秒。对于深度研究，建议增加到 120 秒以上："timeout": 180000
  • 确保配置了论文 API 密钥；缺失密钥会导致论文源不可用，研究无法进行

缓存工具无法使用

  • 确认 .env 中有 PAPER_CACHE_ENABLED=true（默认值）
  • 缓存工具（paper_cache_stats、paper_cache_cleanup）独立于 ENABLE_PAPER_TOOLS

远程连接在项目级配置中不生效

  • OpenCode 已知 issue：MCP 服务器在项目级 opencode.json 中有时无法被识别。
  • 解决方法：将 MCP 配置放在全局配置文件 ~/.config/opencode/opencode.json 中。
